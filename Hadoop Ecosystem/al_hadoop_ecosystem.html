Hadoop Ecosystem Tools

1. HDFS (Hadoop Distributed File System)

কাজ:

HDFS একটি ডিসট্রিবিউটেড ফাইল সিস্টেম যা Hadoop ক্লাস্টারে ডেটা সংরক্ষণ এবং অ্যাক্সেস করতে ব্যবহৃত হয়। এটি ডেটা পার্টিশন করে, ক্লাস্টার জুড়ে ডিস্ট্রিবিউটেড রাখে এবং ব্যাকআপও সরবরাহ করে।

ব্যবহার:

  • hadoop fs -ls /: ফাইল সিস্টেমের বিষয়বস্তু দেখতে সাহায্য করে।
  • hadoop fs -put : লোকাল ফাইল HDFS-এ আপলোড করতে ব্যবহার হয়।
  • hadoop fs -get : HDFS থেকে লোকাল ডিরেক্টরিতে ফাইল ডাউনলোড করতে ব্যবহার হয়।
  • hadoop fs -cat : HDFS ফাইলের বিষয়বস্তু দেখতে সাহায্য করে।
  • hadoop fs -rm : HDFS থেকে ফাইল মুছে ফেলে।
  • hadoop fs -mkdir : HDFS-এ একটি নতুন ডিরেক্টরি তৈরি করতে ব্যবহৃত হয়।
  • hadoop fs -cp : HDFS-এর মধ্যে ফাইল কপি করতে ব্যবহার হয়।
  • hadoop fs -mv : HDFS-এ ফাইল স্থানান্তর করতে ব্যবহার হয়।
  • hadoop fs -du : HDFS পাথের ডিস্ক ব্যবহার পরিমাপ করতে ব্যবহৃত হয়।
  • hadoop fs -tail : HDFS ফাইলের শেষের কিছু অংশ দেখতে সাহায্য করে।
  • hadoop fs -stat : ফাইলের মেটাডেটা দেখতে সাহায্য করে।

2. Hive

কাজ:

Hive একটি ডেটাবেস সফটওয়্যার যা SQL-এর মতো সিনট্যাক্স ব্যবহার করে ডেটা বিশ্লেষণ করার জন্য ব্যবহৃত হয়। এটি ডেটাকে টেবিলের আকারে সঞ্চিত করে এবং ডেটা প্রসেসিং এর জন্য MapReduce এর পিছনে কাজ করে।

ব্যবহার:

  • CREATE DATABASE ;: একটি নতুন ডাটাবেস তৈরি করতে ব্যবহৃত হয়।
  • SHOW DATABASES;: Hive-এর সমস্ত ডাটাবেস দেখতে সাহায্য করে।
  • USE ;: নির্দিষ্ট ডাটাবেস নির্বাচন করতে ব্যবহৃত হয়।
  • CREATE TABLE ( , ...);: একটি নতুন টেবিল তৈরি করতে ব্যবহৃত হয়।
  • SHOW TABLES;: Hive ডাটাবেসের সমস্ত টেবিল দেখতে ব্যবহৃত হয়।
  • DESCRIBE ;: টেবিলের কাঠামো দেখতে ব্যবহৃত হয়।
  • SELECT * FROM ;: একটি টেবিলের সমস্ত ডেটা দেখতে ব্যবহৃত হয়।
  • LOAD DATA INPATH '' INTO TABLE ;: HDFS থেকে ডেটা লোড করতে ব্যবহৃত হয়।
  • INSERT INTO TABLE VALUES (...);: টেবিলে ডেটা ইনসার্ট করতে ব্যবহৃত হয়।
  • SELECT COUNT(*) FROM ;: টেবিলের মোট রেকর্ড গোনার জন্য ব্যবহৃত হয়।
  • ALTER TABLE ADD COLUMNS ( );: টেবিলের নতুন কলাম যোগ করতে ব্যবহৃত হয়।
  • DROP TABLE ;: একটি টেবিল মুছে ফেলতে ব্যবহৃত হয়।
  • SHOW PARTITIONS ;: পার্টিশন দেখানোর জন্য ব্যবহৃত হয়।
  • SELECT DISTINCT FROM ;: ডিস্টিংক্ট মান গুলি নির্বাচন করতে ব্যবহৃত হয়।

3. Sqoop

কাজ:

Sqoop একটি টুল যা রিলেশনাল ডেটাবেস থেকে Hadoop-এর মধ্যে ডেটা ইম্পোর্ট এবং এক্সপোর্ট করার জন্য ব্যবহৃত হয়।

ব্যবহার:

  • sqoop import --connect --table --target-dir : রিলেশনাল ডেটাবেস থেকে HDFS-এ ডেটা ইম্পোর্ট করতে ব্যবহৃত হয়।
  • sqoop export --connect --table --export-dir : HDFS থেকে রিলেশনাল ডেটাবেসে ডেটা এক্সপোর্ট করতে ব্যবহৃত হয়।
  • sqoop list-databases --connect : সমস্ত ডাটাবেসের তালিকা দেখানোর জন্য ব্যবহৃত হয়।
  • sqoop list-tables --connect --database : একটি নির্দিষ্ট ডাটাবেসের টেবিলগুলি দেখতে ব্যবহৃত হয়।
  • sqoop eval --connect --query : SQL কোয়েরি এক্সিকিউট করতে ব্যবহৃত হয়।
  • sqoop create-hive-table --connect --table --hive-table : Hive-এ একটি টেবিল তৈরি করতে ব্যবহৃত হয়।

4. Flume

কাজ:

Flume একটি ডিস্ট্রিবিউটেড, ফ্লেক্সিবল এবং নিরাপদ ডেটা সংগ্রহের টুল যা লগ ডেটা সংগ্রহ এবং HDFS বা অন্যান্য ডেটা স্টোরেজ সিস্টেমে পাঠাতে ব্যবহৃত হয়।

ব্যবহার:

  • flume-ng agent --conf --conf-file --name : Flume এজেন্ট চালাতে ব্যবহৃত হয়।
  • flume-ng start : Flume এজেন্ট চালু করার জন্য ব্যবহৃত হয়।
  • flume-ng stop : Flume এজেন্ট বন্ধ করতে ব্যবহৃত হয়।
  • flume-ng status : Flume এজেন্টের স্ট্যাটাস দেখার জন্য ব্যবহৃত হয়।
  • flume-ng run : Flume কনফিগারেশন চালাতে ব্যবহৃত হয়।

5. Spark

কাজ:

Spark একটি ফাস্ট এবং জেনেরেল পারপাস ক্লাস্টার কম্পিউটিং সিস্টেম যা বিশাল পরিমাণ ডেটা বিশ্লেষণ এবং প্রসেসিং করতে ব্যবহৃত হয়। এটি দ্রুত এবং মেমরি-ভিত্তিক প্রসেসিং করার জন্য বিখ্যাত।

ব্যবহার:

  • spark-submit --class --master : Spark অ্যাপ্লিকেশন চালাতে ব্যবহৃত হয়।
  • spark-shell: Spark এর ইন্টারেক্টিভ শেল চালাতে ব্যবহৃত হয়।
  • spark-submit --deploy-mode cluster --conf : ক্লাস্টারে Spark অ্যাপ্লিকেশন চালানোর জন্য ব্যবহৃত হয়।
  • spark-submit --driver-memory --executor-memory : ড্রাইভার এবং এক্সিকিউটর মেমরি কনফিগার করতে ব্যবহৃত হয়।
